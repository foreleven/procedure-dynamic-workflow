# 金融售前咨询业务 Procedure

本流程用于小微企业融资产品的售前咨询。它只做初步需求澄清、产品匹配、方案说明和销售线索创建，不做授信审批、不承诺额度、不承诺利率，也不收集用户未主动提供的敏感材料。

用户进入咨询时，先读取客户画像、现有业务关系和当前可售产品，分别使用 {@connectors.presales.getCustomerProfile}、{@connectors.presales.getExistingRelationships}、{@connectors.presales.getProductCatalog}。这些信息只作为咨询上下文，不代表审批结果。

用户通常会表达融资金额、资金用途和期望期限。金额和用途是产品匹配的最低必要条件；期限可以用于测算，但不能因为缺少期限就阻止初步推荐。用途包括经营周转、进货备货、设备采购、扩张装修、发票回款等。用户改变金额、用途或期限时，旧的产品选择、测算结果和未创建的转顾问意向都需要重新确认。

当用户给出金额和用途后，通过 {@connectors.presales.matchProducts} 匹配候选产品。系统应说明匹配原因、关键门槛、所需材料和不确定性，不能把候选产品说成已经获批。客户如果明确选择某个产品或询问某个产品详情，可以通过 {@connectors.presales.getProductDetails} 获取产品说明。

如果用户给出或确认产品、金额和期限，可以通过 {@connectors.presales.calculateIndicativeRepayment} 做试算。试算结果只是售前估算，必须说明实际利率、费用和审批结果以正式审核为准。

只有当用户明确要求“销售顾问联系我”“继续办理”“帮我申请/留资”时，才通过 {@connectors.presales.createConsultationLead} 创建销售线索。创建线索前必须已经有明确选中的产品、融资金额和用途；如果没有，应先继续澄清，不能替用户创建线索。

用户提出取消、暂不咨询或停止跟进时，停止当前售前咨询流程，不再创建线索。已经创建的线索代表外部系统中的销售跟进记录，后续修改需求不应自动删除该线索。

## 必须覆盖的业务分支

1. 用户给出金额和用途后，系统能推荐候选产品，但不能创建销售线索或承诺获批。
2. 用户选择产品并给出期限后，系统能读取产品详情并做非绑定试算。
3. 用户修改金额、用途或期限后，旧的产品选择和试算需要重新确认。
4. 用户只问材料、费用、利率或门槛时，系统只能回答咨询问题，不能创建销售线索。
5. 用户明确要求顾问联系或继续办理时，只有产品、金额和用途齐备才创建销售线索。
6. 新企业、小额需求、设备采购、发票回款等不同用途需要匹配不同候选产品或说明不满足门槛。
7. 用户取消或暂停咨询后，当前流程停止推进。

## 明确不支持的行为

本流程不进行贷款申请提交、不做 KYC/KYB 文件收集、不触发征信、不创建正式授信申请、不修改已有贷款、不处理还款或逾期问题。这些事项应进入相邻流程，例如 loan_application、credit_approval 或 repayment_service。
