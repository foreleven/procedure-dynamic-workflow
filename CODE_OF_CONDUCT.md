# Code of Conduct

This project is intended to be a focused, technical, and respectful collaboration space.

## Expected Behavior

- Be direct and constructive when discussing bugs, designs, and tradeoffs.
- Assume public comments may be read by future contributors who lack the current context.
- Keep feedback tied to code, documentation, behavior, reproducibility, or project scope.
- Respect boundaries around private data, credentials, user logs, and production connector details.

## Unacceptable Behavior

- Harassment, threats, insults, or sustained personal attacks.
- Publishing private information, credentials, customer data, or production connector payloads.
- Deliberately derailing technical discussion or repeatedly ignoring maintainer direction.
- Using issues, pull requests, or discussions to pressure maintainers for private support or disclosure of security details.

## Reporting

For ordinary conduct issues, contact the maintainers through the repository once public maintainer contact is configured. If the concern includes a vulnerability, credential, private customer data, or exploit detail, follow `SECURITY.md` instead of posting publicly.

Maintainers may edit, hide, or remove comments; close or lock threads; restrict participation; or escalate to platform moderation when needed to keep the project maintainable and safe.
